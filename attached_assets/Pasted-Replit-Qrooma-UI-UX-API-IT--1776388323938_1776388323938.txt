Replitへ

Qrooma の UI/UX を追加修正してください。
今回は特に、APIキーの取得導線と説明の分かりやすさ を改善したいです。
非ITの人でも迷わず設定できる状態を目指してください。

今回 fix したいこと

1. APIキー取得導線を、Qrooma内の設定画面ではなく「各社の公式ページ」に飛ばす

やりたいことは、Settings で各プロバイダーごとに
	•	OpenAI
	•	Anthropic
	•	Google

それぞれに対して、APIキー取得ページへ飛ぶボタンを置くことです。

仕様
	•	Qrooma 内で APIキーを自動取得する機能は不要
	•	ボタンを押したら、各社の公式 APIキー取得 / 管理ページ に遷移する
	•	新しいタブで開いてよい
	•	その先でログインが必要な場合、それは各社側の通常フローとして問題なし
	•	そこから先の認証は Qrooma のスコープ外

文言
	•	ja: APIキーを取得
	•	en: Get API key

置き場所
	•	各 provider セクション内
	•	できれば provider 名の近く
	•	「保存」導線と混ざらないように、主CTAより弱い補助導線として配置

⸻

2. 非ITの人にも分かる説明にする

今のAPIキー説明は、ITに慣れていない人には難しい可能性があります。
専門用語を減らして、日常語に近い説明にしてください。

伝えたいこと
	•	Qrooma で議論を動かすには、各AIサービスの利用許可が必要
	•	そのために、自分の OpenAI / Anthropic / Google の APIキーを設定する
	•	APIキーは「そのAIサービスを使うための利用キー」のようなもの
	•	Qrooma はそのキーを使って、あなたの代わりにAIへ依頼する
	•	料金は各サービスの契約に基づいて発生する
	•	キーは安全に保存される
	•	キーがないと議論は実行できない

日本語の説明トーン
難しくしすぎず、たとえば以下の方向で
	•	QroomaでAIに議論してもらうには、各AIサービスの利用キー(APIキー)が必要です。
	•	APIキーは、そのサービスを使うための「利用許可キー」のようなものです。
	•	一度設定すると、Qrooma からそのAIを使って議論を実行できるようになります。

4. 「キーがないと動かない」を明確にする

Qrooma は BYOK 前提なので、APIキー未設定では議論を実行できない ことを明確にしてください。

UIで必要なこと
	•	API Key 未設定バッジは維持
	•	ただし警告しすぎず、分かりやすい表示にする
	•	必要なキーがない場合は、
	•	なぜ動かないか
	•	どこで設定するか
が分かるようにする

日本語文言イメージ
	•	このAIを使うにはAPIキーの設定が必要です
	•	まずは「APIキーを取得」から設定してください
