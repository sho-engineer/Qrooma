Replitへ

Qrooma のプラン別UIを見直してください。
今の Free モードは、機能制限だけ入っていて、UIがプランごとに最適化されていないため、導線が分かりづらいです。
今回は plan-aware UI にしてください。

問題認識

現在の Free モードは、
	•	Free バッジは出ている
	•	ただし Connect/BYOK 前提の匂いが残っている
	•	3モデル前提の見た目が残っている
	•	APIキー設定導線も近くに見えてしまう
ので、ユーザーにとって
「Freeで何ができるのか」「何が不要なのか」
が直感で分かりません。

今回は、機能制限だけではなく、表示自体をプランごとに変えるようにしてください。

⸻

固定したいプラン挙動

1. Free
	•	APIキー不要
	•	体験版
	•	2エージェントのみ
	•	モデルは固定、または無料対象のみ
	•	APIキー入力UIは表示しない
	•	設定画面は Free 向けの簡易表示にする
	•	Free では「試せる内容」に集中させる

Free の画面で見せたいこと
	•	Freeモード
	•	2エージェント
	•	固定モデルで体験できます
	•	無料枠には利用制限があります
	•	本格利用は Connect / Pro

Free で見せないもの
	•	OpenAI / Anthropic / Google のAPIキー入力欄
	•	3エージェント前提のUI
	•	高度なモデル選択UI
	•	「設定で入力する」前提の主導線

⸻

2. Connect
	•	APIキー必要
	•	2 / 3 エージェント切替可
	•	モデル選択可
	•	APIキー取得導線を表示
	•	本格利用向け

Connect で見せるもの
	•	APIキー入力
	•	APIキーを取得
	•	2/3エージェント切替
	•	モデル選択
	•	保存ボタン

⸻

3. Pro
	•	APIキー不要
	•	一定量込み
	•	超過分は従量課金
	•	Connect ほど API設定色は出さない
	•	Pro 専用の説明を見せる

Pro で見せたいこと
	•	Pro
	•	APIキー不要
	•	月額に一定量の利用を含みます
	•	超過分は従量課金
	•	2/3エージェントやモデル設定は、必要なら一部見せてもよい

Pro で見せないもの
	•	APIキー入力欄
	•	APIキー取得導線

⸻

重要な設計方針

プランごとに表示を切り替える

単に disabled にするだけではなく、表示自体を変えること。

例
	•	Free の room detail では、最初から 2エージェントだけ表示
	•	Free の settings では APIキーセクションを出さない
	•	Connect では API セクションを出す
	•	Pro では API セクションを出さず、利用枠説明を出す

⸻

Room detail の表示ルール

Free
	•	2エージェントだけ表示
	•	役割表示も 2人分だけ
	•	提案（GPT）
	•	検証（Claude）
のように表示してよい
	•	3人目は出さない
	•	Freeモード・2エージェント・固定モデルで体験できます
のような補助説明を自然に出す

Connect
	•	2/3 エージェント設定に従う
	•	役割 + モデル表示
	•	APIキー未設定の provider がある場合はその前提を出す

Pro
	•	2/3 エージェント設定に従う
	•	APIキー不要であることを自然に伝える
	•	利用枠の説明を必要に応じて表示する

⸻

Settings の表示ルール

Free

Settings は簡易版にしてください。

見せたい内容:
	•	現在のプラン: Free
	•	無料枠には利用制限があります
	•	2エージェント / 固定モデル
	•	Connect / Pro への案内

見せない内容:
	•	APIキー入力欄
	•	各providerの APIキーを取得
	•	高度なモデル設定

Connect

Settings はフル版でOK。
	•	APIキー入力
	•	APIキー取得導線
	•	モデル設定
	•	2/3エージェント
	•	保存ボタン

Pro

Settings は中間版にしてください。
	•	APIキー入力は非表示
	•	利用枠説明を表示
	•	必要なら 2/3 エージェントや一部モデル設定は可
	•	保存ボタンあり

LP / 料金導線の見せ方も合わせる

料金表・説明文も、今回のUIに合わせて整合させてください。

Free
	•	APIキー不要
	•	まず試せる
	•	無料枠には利用制限あり

Connect
	•	自分のAPIキーで使う
	•	モデルや構成を選べる

Pro

	•	利用込み
	•	超過従量

テストケースでの特別対応

本番では見せなくてよいですが、今回のテストではプラン切替を容易にしてください。

要望
	•	開発用の簡易プラン切替UIを用意
	•	Free / Connect / Pro をすぐ切り替えられるようにする
	•	切り替えると、各画面の表示もそれに応じて変わること
	•	これは本番では hidden / disabled / dev only でよい

目的
	•	画面ごとの差をすぐ確認したい
	•	Free / Connect / Pro の導線を見比べたい
	•	本番実装前に UX を固めたい
