Qrooma に、議論開始前の確認フローを実装してください。
目的は、曖昧な入力のまま議論を始めて方向がズレるのを防ぐことです。

⸻

背景

現状のQroomaは、複合質問や条件依存の強い相談でも、そのまま議論を始めることがあります。
その結果、

* AIが勝手に前提を置く
* 途中で議論の焦点がズレる
* 一部の論点だけ深掘りしてしまう
* ユーザーが本当に欲しかった結論から外れる

ことがあります。

特に以下のようなテーマでは、前提確認が重要です。

* 旅行
* 購入相談
* 比較検討
* 計画づくり
* 人選 / 採用
* 優先順位のある判断

⸻

目的

Qrooma に、必要なときだけ AI側から短く確認するステップを追加してください。

重要なのは、

* 毎回必ず質問することではない
* 曖昧さが大きいときだけ確認する
* ユーザーが「そのまま始める」ことも選べる
    ことです。

⸻

実装したい基本仕様

1. 入力後にまず曖昧さを判定する

ユーザーがメッセージを送信したら、いきなり議論を始める前に、内部で以下を判定してください。

* 条件が十分か
* 複数解釈がありうるか
* 前提次第で結論が大きく変わるか
* 優先順位が不明か
* 制約条件が不足しているか

判定結果

* 十分に明確 → そのまま議論開始
* 曖昧さが高い → 確認フローを出す

⸻

2. 確認フローのUI

曖昧と判定した場合、議論を始める前に、短い確認ステップを表示してください。

表示例

議論を始める前に、少しだけ確認したいことがあります。

* 旅行時期はいつですか？
* 子ども連れですか？
* 今回いちばん重視したいのは、食事・景色・移動の楽さのどれですか？

ルール

* 最大3問まで
* 長くしない
* 質問は簡潔に
* 優先度の高いものだけ聞く

⸻

3. ユーザーの選択肢

確認フローの下に、次の2つの導線を出してください。

* 回答してから始める
* このまま議論する

A. 回答してから始める

* ユーザーが回答を入力
* 回答を前提条件に追加
* その条件で議論開始

B. このまま議論する

* 確認未回答のまま開始する
* ただし、AIは内部で仮置きした前提を明示する

⸻

4. 「このまま議論する」時のルール

この導線を選んだ場合、AIは勝手に前提を置いてもよいが、何を仮置きしたか必ず明示してください。

例

* 「旅行時期が未指定のため、季節依存の少ない候補を中心に議論します」
* 「予算条件が未指定のため、標準的な価格帯を前提に考えます」
* 「お子さまの年齢が不明なため、家族全体が無理なく楽しめる案を優先します」

重要

前提を黙って置かないこと。
仮置き条件は、議論開始前または Round 1 冒頭で短く明示してください。

⸻

5. 確認が必要なテーマの判定ルール

以下のようなケースでは、確認フローを出しやすくしてください。

確認推奨のケース

* 条件が複数あり、優先順位が不明
* 日時 / 季節 / 予算 / 人数 / 年齢で結論が大きく変わる
* 地域や候補が広い
* 比較軸が見えない
* 「おすすめ教えて」系で条件が少ない

例

* 「家族旅行どこがいい？」
* 「おすすめのPC教えて」
* 「転職するべき？」
* 「どっちのプランがいい？」

⸻

6. 確認が不要なケース

以下では、確認を出さずそのまま議論してよいです。

* ユーザーがラフな壁打ちをしている
* 多少ズレても後で修正しやすい
* 明確に案出しだけが目的
* ユーザーが「ひとまず議論して」と言っている
* 入力情報が十分ある

⸻

7. 内部実装の考え方

追加したい内部状態

* needsClarification: boolean
* clarificationQuestions: string[]
* clarificationAnswers: object
* assumptions: string[]
* debateStartMode: "clarified" | "assumed"

フロー

1. message submit
2. ambiguity check
3. if clear → debate start
4. if unclear → clarification UI
5. user chooses
    * answer first
    * proceed anyway
6. debate starts with clarified conditions or explicit assumptions

⸻

8. 表示文言

日本語

* 議論を始める前に、少しだけ確認したいことがあります
* 回答してから始める
* このまま議論する
* 仮置きした前提
* 未指定の条件があるため、以下の前提で議論を始めます

英語

* Before we start, I’d like to confirm a few things
* Answer first
* Start anyway
* Assumptions used
* Since some conditions were not specified, the discussion will start with the following assumptions

⸻

9. UX要件

重要

確認フローは、テンポを壊さないようにしてください。

* 毎回出さない
* 長い質問リストにしない
* 2〜3問に絞る
* ユーザーがスキップできる

理想の体験

* 明確な質問なら即議論開始
* 曖昧ならAIが少し確認
* ユーザーは答えるかスキップするか選べる
* スキップしたら前提を明示して議論開始

⸻

10. 受け入れ条件

以下を満たしたらOKです。

機能

* 曖昧な入力で確認フローが出る
* 明確な入力ではそのまま始まる
* ユーザーが回答してから始めるか、そのまま始めるか選べる

表示

* スキップ時に仮置き前提が明示される
* どの前提で議論したかが見える

UX

* 質問は最大3つまで
* 冗長でない
* 議論開始のテンポを壊さない
