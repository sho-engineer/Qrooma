Qrooma を以下の内容で修正してください。
今回は、議論品質の改善よりも、アプリとしての基本UXとエラーハンドリングの品質向上を優先してください。

今回の最優先課題

1. Gemini の raw error がそのまま画面に出ている
2. モバイルで改行キーを押すと送信されてしまう
3. 入力中 / 生成中に停止できない
4. モデル名ラベルが実際のモデルとズレている
5. サイドバーから削除確認後、一覧から消えない
6. 比較軸に「その他」がない

⸻

1. rawなベンダーエラーをそのまま表示しない

現状

画面に以下のような vendor error がそのまま表示されている。

* GoogleGenerativeAI Error
* 429 Too Many Requests
* quota exceeded
* raw JSON / raw API message

これはユーザー向けUIとして不適切です。

必須対応

A. UIでは raw error を出さない

ユーザー画面には、以下のような人向けの短い文言だけを表示してください。

例:

* 現在このAIが混み合っています。少し時間をおいて再試行してください。
* 無料枠の上限に達したため、このAIは一時的に利用できません。
* 一部のAIで応答できなかったため、残りのAIで続行しました。

B. 詳細エラーは内部ログだけに残す

raw error は

* console
* server log
* debug log
    には残してよいが、通常UIには出さない

C. エラー種別ごとの出し分け

最低限以下を判定して、文言を変えること。

* quota exceeded / rate limit
* timeout
* auth error
* unknown provider error

D. Freeプラン時の挙動

Free で Gemini 側だけ落ちた場合、

* 可能なら残りの応答で暫定まとめを出す
* 無理なら「一部のAIが利用できず、完全な議論を生成できませんでした」と出す
* 少なくとも raw error は出さない

期待挙動

* 赤い長文エラーが出ない
* 代わりに短い自然なメッセージになる
* 必要なら再実行ボタンを案内する

⸻

2. モバイルでは改行キーで送信しない

現状

改行したいのに Enter / 改行キーで送信されてしまう

必須対応

モバイル

* 改行キー = 改行のみ
* 送信は送信アイコン押下のみ

デスクトップ

* Enter送信 / Shift+Enter改行のままでもよい

文言修正

モバイルで以下のような文言があるなら修正する

* Enterで送信、Shift+Enterで改行

モバイルでは以下に変更

* メッセージを入力
    または
* 改行できます。送信は右のボタンから

⸻

3. 生成中に停止できるようにする

現状

入力中 / 生成中に途中停止できない

必須対応

生成中は、送信ボタンの代わりに 停止ボタン を表示してください。

仕様

状態

* idle
* generating
* stopping

generating中

* 入力欄右端は送信アイコンではなく 停止 アイコンにする
* ラベルは不要でもよいが、押して意味が分かる見た目にする

停止時

* 進行中の stream / provider call を abort する
* 状態を stopped または partial として保存する
* 途中まで出ている内容は残す
* まとめ / 結論は
    * 生成可能なら暫定版を出す
    * 無理なら「途中で停止しました」と出す

期待挙動

* ユーザーが途中で止められる
* アプリが固まらない
* 中途半端な状態でも破綻しない

⸻

4. モデル名ラベルのズレを直す

現状

* Gemini を使っているはずなのに Claude と出る
* 実際の provider / model と表示名がズレている

必須対応

表示ラベルは、実際にそのメッセージを生成した provider / model から一貫して決めること。

確認したい点

* message object に provider / model / role が正しく保存されているか
* UIで別の run の metadata を参照していないか
* fallback時にラベルだけ古いまま残っていないか

期待挙動

* GPT の発言は GPT 表示
* Gemini の発言は Gemini 表示
* Claude の発言は Claude 表示
* fallbackした場合は、その実際の provider 名に変わる

⸻

5. サイドバー削除後に一覧から消えるようにする

現状

* 削除しますか？ は出る
* でもその後、一覧から消えない / 反映が遅い / 状態がズレる

必須対応

A. 削除後のUI更新

削除成功後は、サイドバー一覧から即座に消してください。

B. 現在開いているルームを削除した場合

* 適切な別ルームへ遷移
    または
* 空状態画面へ遷移

C. optimistic update でもよい

* 先に一覧から消す
* 失敗したら復元

期待挙動

* 削除後に残らない
* 二重表示や空白が出ない
* 次のルーム選択状態も破綻しない

⸻

6. 比較軸に「その他」を追加

現状

比較軸の候補が固定で、ユーザーが独自の軸を入れにくい

必須対応

比較軸選択に 「その他」 を追加してください。

仕様

* その他 を選ぶと自由入力欄が開く
* 複数入力できてもよい
* 内部では comparisonAxes に追加保存する
* 議論ロジックにも反映する

プレースホルダー例

* 自分で比較軸を入力
* 例: 初回旅行向き / 子どもの満足度 / 写真映え / 雨耐性

期待挙動

* 定義済み比較軸にないものも使える
* それが summary / conclusion に反映される

⸻

7. 補足改善

A. エラー時の結論欄

provider error が起きても、結論欄を壊さない

* 一部の応答で問題が発生しました
* 再実行するか、別条件で試してください
    のような短い表示にする

B. Freeプランの制限文言

quota超過時は、必要なら

* Freeプランの利用上限に達しました
* 時間をおいて再試行してください
    のように、プラン文脈に合う案内も検討する

C. 再実行ボタン

エラー後も再実行導線が自然に使えるようにする

⸻

以下を満たしたらOKです。

* 削除したルームが残らない

比較軸

* 「その他」が使える
* 自由入力した軸が議論に反映される
