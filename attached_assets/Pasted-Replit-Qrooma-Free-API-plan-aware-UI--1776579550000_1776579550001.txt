Replitへ

Qrooma に以下の修正を入れてください。
今回は アーカイブ済みトークの閲覧導線追加 と、Free プランでは API キー関連を完全に隠す plan-aware UI が目的です。

1. アーカイブ済みトークを見られる場所を作る

現状だと、アーカイブしたルームをどこで見ればいいか分かりません。
/rooms の中で、通常一覧 と アーカイブ一覧 を切り替えられるようにしてください。

期待する挙動

* 通常のルーム一覧とは別に、アーカイブ済み を見られる
* 切替方法は以下のどちらかでOK
    * 一覧上部に すべて / アーカイブ タブ
    * サイドバーに アーカイブ済み セクション
* アーカイブ済みルームでは、最低限
    * 戻す
    * 削除
        ができるようにする
* 通常一覧にはアーカイブ済みルームを出さない

文言

* ja: アーカイブ済み, 戻す, 削除
* en: Archived, Restore, Delete

注意

* まずは UI とローカル状態でもよい
* 後で本実装に接続しやすい構造にする
* モバイルでも切替しやすくする

⸻

2. Free プランでは API キー関連を表示しない

Free は APIキー不要 の前提なので、ユーザー向けUIに APIキー入力欄や API取得導線を出さないでください。

Free の方針

* APIキー入力欄は 非表示
* APIキーを取得 リンクも 非表示
* provider ごとのキー状態表示も 非表示
* 高度なモデル設定も 非表示
* Free では「試せる内容」にだけ集中させる

Free で見せたい内容

* 現在のプラン: Free
* APIキー不要で試せます
* 無料枠には利用制限があります
* 2エージェント / 固定モデルで体験できます
* 本格利用は Connect / Pro

Free の Settings は簡易版にする

見せるもの:

* 現在のプラン
* 無料枠の説明
* 使える範囲の説明
* Connect / Pro への案内

見せないもの:

* OpenAI / Anthropic / Google の API キー欄
* APIキーを取得
* モデルの詳細な切替
* 3エージェント前提の設定

⸻

3. Free 用のキーはユーザー設定ではなく、サーバー管理であることを前提にする

重要な仕様として、Free 用の API キーはユーザーが設定するものではない です。

前提

* Free では Qrooma 側が持つサーバー側キーを使う
* ユーザーは Free で API キーを入力しない
* Connect のときだけユーザーが自分の API キーを設定する
* Pro でも API キー入力は不要

UIに反映したいこと

ユーザーに見える説明は例えばこうです。

日本語

* FreeではAPIキー不要で試せます
* Connectでは、自分のAPIキーで本格利用できます
* ProではAPIキー不要で、そのまま利用できます

英語

* Free works without API keys
* Connect lets you use your own API keys
* Pro works without API keys and includes usage

⸻

4. プランごとの表示をさらに明確にする

Free

* APIキー関連を表示しない
* 2エージェントのみ表示
* 固定モデル前提
* 体験用導線に集中

Connect

* APIキー関連を表示する
* APIキーを取得
* 2/3 エージェント切替
* モデル選択
* 保存ボタンあり

Pro

* APIキー関連は表示しない
* 利用込みプランである説明を出す
* 必要なら一部設定のみ表示
