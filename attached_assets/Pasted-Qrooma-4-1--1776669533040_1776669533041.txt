Qrooma を以下の内容で修正してください。
今回は 議論品質の改善より先に、体験を壊している不具合と状態遷移を直すこと を優先してください。

⸻

目的

以下の4点を修正してください。

1. 日本語UIなのに途中から英語が混ざる
2. 2ラウンド終わっても結論が出ないことがある
3. 結論が出ない場合の導線がない
4. 2ラウンド後にチャット入力がフリーズすることがある

⸻

1. 日本語UIで英語が混ざる不具合を直す

現象

* 日本語で質問している
* UIも日本語
* なのに途中から Gemini / rebuttal / critique が英語になることがある

原因候補

* validator / critique prompt が英語ベース
* rebuttal step に言語設定が引き継がれていない
* internal debate logic の中間出力をそのまま表示している
* summary/conclusion 前の中間データを未翻訳で出している

対応

* UI言語が日本語のときは、全ラウンド・全エージェント・全summary・全conclusion を日本語固定
* 内部的に英語テンプレや英語中間データを使ってもよいが、表示前に必ず日本語化
* 表示直前に language guard を入れる
* 日本語モードで英語文率が一定以上なら、そのまま表示せず再整形する

期待挙動

* 日本語モードでは英語が1文も混ざらない
* 特に rebuttal / critique / round summary / conclusion で混在しない

⸻

2. 結論が出ない問題を直す

現象

* Round 2 まで進む
* その後、結論が 生成中 のまま / エラー / 空表示
* なのに run 全体は完了扱いになっていることがある

認識

これは「議論本体」と「結論生成」が別処理なのに、
finalizer が落ちても run を完了にしている可能性が高いです。

対応

A. 状態を分離する

以下の状態を分けて持つようにしてください。

* runStatus:
    * running
    * awaiting_conclusion
    * awaiting_continue
    * completed
    * error
* conclusionStatus:
    * idle
    * generating
    * ready
    * unresolved
    * error

B. 完了条件を修正する

* run completed は「結論 ready」または「未確定として処理完了」まで行ってからにする
* Round 2 終了だけで completed にしない

C. 結論カードの状態表示

結論カードは次の状態を持たせる

* generating: 「結論を生成中…」
* ready: 結論本文表示
* unresolved: 「まだ結論は固まっていません」
* error: 「結論を生成できませんでした」
* idle/empty: 空欄表示しない

⸻

3. 結論が出ない場合の導線を追加する

今は、結論が出ないと空欄やエラーで終わってしまう。
これは不自然なので、未収束状態を正式な状態として扱ってください。

未収束時の表示

結論が固まらない場合は、結論カードに以下を表示する

* タイトル: 結論はまだ未確定です
* 補足: 争点が残っているため、この時点では確定結論を出していません

ボタン

以下のアクションを出してください。

* 議論を続ける
* 今の内容で暫定結論を出す
* 条件を追加する

ボタン仕様

議論を続ける

* 次ラウンドを追加実行
* 既存の文脈と争点を引き継ぐ

今の内容で暫定結論を出す

* 完全収束ではなくても、現時点での最善案を結論化する

条件を追加する

* 入力欄を有効にしたまま
* 「人数」「予算」「制約」など前提を足して再開できる

⸻

4. 2ラウンド後にチャットがフリーズする問題を直す

現象

* 2ラウンド後、見た目はほぼ終わっている
* でも入力欄が disabled のまま
* チャットが再開できない

原因候補

* isStreaming
* isGeneratingConclusion
* isRunLocked
    などのフラグが、error/unresolved 時に解除されていない

対応

A. 入力欄 disable 条件を見直す

入力欄を disable にするのは、
本体のラウンド生成中だけにしてください。

B. 結論生成失敗ではフリーズさせない

* 結論が error でも入力欄は再開
* 結論だけ再試行できる
* unresolved でも入力欄は再開

C. 明示的に解除する

以下のどちらかで run が一区切りしたら、必ず input を reopen する

* conclusionStatus = ready
* conclusionStatus = unresolved
* conclusionStatus = error

期待挙動

* 結論失敗でも新しい入力はできる
* 結論未確定でも「続ける」「条件追加」が押せる
* チャット全体が固まらない

⸻

5. 追加で修正してほしいこと

A. 日本語ラベルを自然にする

以下のように揃えてください

* 結論 生成中 → 結論を生成中...
* 結論 エラー → 結論を生成できませんでした
* 空欄時 → 実行完了後に表示されます ではなく、状態に応じて出し分け

B. Round summary と Conclusion の整合

* Round 2 暫定まとめで「未解決の核心要望が残る」と書いたなら、
    結論カードでも 未確定 or 暫定結論 に繋がるようにする
* summary と conclusion の状態が矛盾しないようにする

C. Continue系の履歴管理

* 議論を続ける を押したら Run を増やす
* Conclusion history にも反映する
* 最新結論と履歴の整合を取る

⸻

6. 実装の考え方

優先順位

1. 言語固定
2. 結論生成の完了保証
3. 未収束時の continue 導線
4. 入力フリーズ解消

重要

これはモデル性能の問題ではなく、オーケストレーションと状態管理の問題として修正してください。
モデルを変えても、この状態管理が直らない限り体験は改善しません。

⸻

7. 受け入れ条件

以下を満たしたらOKです。

言語

* 日本語UIで英語混在が起きない

結論

* 2ラウンド後に必ず以下のどれかになる
    * 結論表示
    * 暫定結論表示
    * 未確定表示 + 続行導線
    * エラー表示 + 再試行導線

チャット

* 結論エラーでもフリーズしない
* 結論未確定でも入力できる
* 続ける / 条件追加ができる
