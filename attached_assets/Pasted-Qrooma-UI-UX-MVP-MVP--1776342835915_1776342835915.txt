Qrooma の UI/UX をまとめて修正してください。
今回は 新機能を広げる のではなく、MVPとしての完成度を上げる ことが目的です。

まず固定したい前提

MVPでやること
	•	テキスト入力中心の体験を磨く
	•	ランディングページとアプリ本体の完成度を上げる
	•	2 / 3 エージェント切替
	•	同一モデル重複禁止
	•	APIキー導線の分かりやすさ改善
	•	モバイルを含めた見た目の質感向上

MVPでやらないこと
	•	画像アップロード
	•	ファイルアップロード
	•	添付ファイルプレビュー
	•	OCR / ファイル解析
	•	画像やPDFをAIに渡す処理

理由：
Qrooma のMVPは、まず 「複数の視点で議論して結論を返す」 体験を成立させることが最優先です。
画像やファイル添付は、ストレージ・解析・セキュリティ・UI複雑化を伴うため、今回はスコープ外にしてください。

⸻

デザインの方向性

ゴール
	•	“AIサービスっぽい派手さ” を減らす
	•	“白に黒を置いただけ” の単調さもなくす
	•	静かで上質、でも平坦ではない
	•	Appleっぽい空気感 + 実用的なB2B SaaS
	•	“生成AIのLP” ではなく、思考支援プロダクト に見せる

重要な方針
	•	色数は少なく
	•	でも階層はしっかり作る
	•	タイポ、面の差、余白、軽い動きで強弱を出す
	•	モデル名やAI感を前面に出しすぎない
	•	情報の主役は「議論」と「結論」

⸻

直したい問題

1. 階層が弱い
	•	文字サイズ、太さ、色が均一で、白に黒を乗せただけに見える
	•	Hero / section title / card title / body / helper の差をもっと明確にする

2. プレースホルダー感が強い
	•	ランディングの product preview が skeleton / ダミー箱っぽい
	•	“何も表示されていない” ようにも見える

3. 動きが少ない
	•	静かにしたいが、止まりすぎている
	•	少しだけ気持ちいい motion を入れたい

4. モバイルで縦に長すぎる
	•	Rooms 一覧カード
	•	Settings の各サイド設定カード
	•	使い方セクション
	•	各カードの余白が大きすぎて間延びしている

5. 仕様漏れ
	•	同じ provider + model の重複禁止が効いていない
	•	2 / 3 エージェント切替が表示されていない
	•	横はみ出しが再発していないか不安

⸻

デザイン修正方針

A. Typography hierarchy を強くする

最低限、以下の階層を明確に分けてください。
	•	Hero
	•	セクション見出し
	•	カード見出し
	•	本文
	•	補助文 / ラベル

サイズ、weight、行間、余白で差を作ってください。
色でごまかさず、タイポで強弱 を出したいです。

B. Surface layering を入れる

白系の中で面の差を作ってください。
	•	page background
	•	section background
	•	card background

全部同じ白にしないで、静かに深さがある 状態にしてください。

C. テキストカラーを3階層にする
	•	primary text
	•	secondary text
	•	tertiary text

見出し・本文・補助文が全部同じ黒や灰色に見えないようにしてください。

D. Product preview を作り直す

ランディングのプレビューは skeleton をやめて、Qrooma の実UIを抽象化したモック に変えてください。

要件：
	•	左にルーム一覧
	•	右に議論
	•	下に結論カード
	•	上部に静かなヘッダー
	•	“完成したプロダクトの一部” に見えること

禁止：
	•	骨組みだけの灰色箱
	•	ローディング画面みたいな見た目

E. Motion を少し入れる

派手な動きは禁止です。
以下くらいの、静かな動きを入れてください。

入れてよいもの：
	•	Hero / cards / preview の軽い fade-up
	•	hover 時のごく小さい浮き
	•	CTA の軽い opacity / transform
	•	アコーディオンの自然な開閉

禁止：
	•	バウンス
	•	強いズーム
	•	ネオン演出
	•	サイバー感

F. モバイル密度を改善する

モバイルで縦に長すぎる箇所を詰めてください。

対象：
	•	Rooms 一覧カード
	•	Settings カード
	•	使い方セクション
	•	Landing の各情報カード

方向：
	•	padding を減らす
	•	高さを減らす
	•	でも窮屈にはしない
	•	“余白” と “間延び” を分ける

⸻

機能 / UI の修正

1. 2 / 3 エージェント切替

UI上に明確に表示してください。

仕様：
	•	2 または 3
	•	default は 3
	•	2 のときは Side A / B のみ表示
	•	3 のときは Side A / B / C を表示

文言：
	•	ja: 参加エージェント数
	•	en: Number of agents

2. 同一モデルの重複禁止

同じ provider + model の組み合わせは重複不可 にしてください。

要件：
	•	他サイドで使われている組み合わせは disabled
	•	少なくとも保存時エラーでも可
	•	ただし今回はできれば選択時点で防ぎたい

エラーメッセージ：
	•	ja: 同じモデルは複数の枠に設定できません
	•	en: The same model cannot be assigned to multiple sides

3. 横はみ出し防止

以下すべてで再確認してください。
	•	Landing
	•	Rooms
	•	Room detail
	•	Settings
	•	Sidebar
	•	モバイルドロワー

overflow-x-hidden で隠すだけではなく、レイアウト自体を収める ことを優先してください。

4. 使い方セクション

今の大きなカードではなく、「① APIキーを設定」型 に変えてください。

表示したい内容：
	•	① APIキーを設定
	•	② ルームを作成
	•	③ テーマを投げて、議論と結論を見る

方向：
	•	番号が主役
	•	短い見出しを強く
	•	補助説明は最小限
	•	モバイルで縦長になりすぎない

5. APIキー導線

Settings では、BYOK の説明をもっと自然に見せてください。
	•	“警告” より “理解しやすい説明”
	•	未設定バッジは残してよいが、静かな表現に
	•	APIキー取得方法への導線は分かりやすく

⸻

ページごとの方向

Landing
	•	Hero の完成度を上げる
	•	product preview を本物っぽくする
	•	使い方を軽くする
	•	セクション間の強弱を作る

Rooms
	•	一覧をもっとコンパクトに
	•	ルームカードの高さを下げる
	•	タイトル / プレビュー / 状態の階層を明確に

Settings
	•	フォームUIの完成度を上げる
	•	サイドA/B/C カードの縦長感を減らす
	•	2/3 切替を見えるようにする
	•	重複禁止を効かせる

Room detail
	•	色を減らし、タイポと余白で見せる
	•	結論カードを特に上質に
	•	“AIチャット” より “議論の場” に見せる
