Replitへ

追加修正です。
今回は以下の3点を優先して直してください。

1. 「保存し」表示崩れを直す

設定画面で 保存し のように文字が途中で切れています。
これは見た目ではなく UI実装の不具合 として直してください。

期待する状態

* 保存する が全文表示される
* 日本語 / 英語どちらでも切れない
* モバイルでも崩れない
* ボタンやバッジの中央揃えが崩れない

確認してほしいこと

* truncate
* overflow-hidden
* whitespace-nowrap
* min-width
* flex-shrink
* 親要素の display:flex / gap
* ボタン幅と padding 不足

方針

* 文字を隠してごまかさない
* ラベルの全文が見えることを優先
* 保存する / Save changes の両方で確認

⸻

2. ルームを「削除」または「アーカイブ」できるようにする

ルーム一覧で、スワイプ操作から削除 / アーカイブを選べるUI を追加してください。

やりたいこと

* ルームカードを横にスワイプ
* アクションが出る
* アーカイブ
* 削除
    を選べる

仕様

アーカイブ

* ルームは通常一覧から外れる
* 完全削除ではない
* 将来的にアーカイブ一覧から戻せる余地を残す

削除

* 完全削除扱い
* 確認ステップがあってもよい
* 少なくとも誤操作しにくくする

UI方針

* モバイルで自然なスワイプ操作
* iOSライクで静かな見た目
* 派手すぎない
* 右スワイプ or 左スワイプは実装しやすい方でよいが、直感的に
* Desktop では ... メニューや hover アクションでも補完してよい

文言

* ja: アーカイブ, 削除
* en: Archive, Delete

注意

* Free / Connect / Pro どのプランでも使える基本機能として扱ってよい
* まずは UI とローカル状態でもよいが、後で本実装に接続しやすい構造にする

⸻

3. APIキー入力後、会話開始しても何も出ない不具合を調査して直す

現在、APIキーを入れたあとで会話を始めても、何も返ってこないようです。
おそらく裏でエラーが起きています。
原因の切り分けと、ユーザーに見えるエラー表示 を入れてください。

まず確認したいこと

* APIキー保存自体は成功しているか
* 保存後に実際の呼び出しでキーが参照されているか
* 必要な provider のキーが揃っているか
* Free / Connect / Pro の分岐が実行時に正しく働いているか
* SSE が繋がっているか
* サーバー側で例外が落ちていないか
* OpenAI / Anthropic / Google SDK 呼び出しで失敗していないか
* エラーが UI に出ずに握りつぶされていないか

期待する状態

* 実行できるときは、会話開始後にちゃんと返答が出る
* 実行できないときは、何が足りないか / なぜ失敗したか を表示する
* “無反応” をやめる

UIで出したいエラー例

* この構成では必要なAPIキーが不足しています
* AIの呼び出しに失敗しました
* 接続に失敗しました。もう一度お試しください
* Freeモードではこの設定は利用できません

開発向け

* ブラウザ console
* network
* server logs
* SSE接続状態
* 各 provider 呼び出しログ
    を見て、どこで止まっているか切り分けてください

実装希望

* サーバー側エラーをUIに返す
* UI側でローディングのまま止まらない
* minimumでも error banner / toast / inline message を出す
* dev時は原因が追えるようにする

継続方針

以下は前回までの方針を維持してください。

* Free / Connect / Pro で表示内容を変える
* Free では APIキー入力UIを出さない
* Connect では APIキー設定とモデル設定を出す
* Pro では APIキー不要の説明を出す
* role + model 表示
* 右端のはみ出し修正
* Appleっぽい静かな見た目
* 保存ボタンあり
* LP の軽いモダンなグラデーション
