Replitへ

Qrooma の UI/UX をまとめて修正してください。
今回は MVPとしての完成度を上げること と、もっとプロフェッショナルで上質な見た目にすること が目的です。

まず固定したい前提

MVPでやること
	•	テキスト入力中心の体験を磨く
	•	ランディングページとアプリ本体の完成度を上げる
	•	Free / BYOK のハイブリッド構成を前提にする
	•	2 / 3 エージェント切替
	•	同一モデル重複禁止
	•	APIキー導線の分かりやすさ改善
	•	モバイルを含めた見た目の質感向上
	•	チャット中の表示をロール名ベースにする
	•	自動結論 + 手動での結論生成導線を入れる
	•	Settings に保存ボタンを入れる

MVPでやらないこと
	•	画像アップロード
	•	ファイルアップロード
	•	添付ファイルプレビュー
	•	OCR / ファイル解析
	•	画像やPDFをAIに渡す処理

理由：
Qrooma のMVPは、まず 「複数の視点で議論して結論を返す」 体験を成立させることが最優先です。
画像やファイル添付は、ストレージ・解析・セキュリティ・UI複雑化を伴うため、今回はスコープ外です。

新しい利用前提

Free
	•	APIキー不要
	•	まずは体験用
	•	すぐ試せる
	•	制限あり
	•	モデル選択不可
	•	2エージェントまでを基本想定
	•	Qrooma 側の固定構成で動く前提
	•	価値を体験するためのモード

BYOK
	•	APIキー必要
	•	本格利用向け
	•	2 / 3 エージェント選択可能
	•	モデル選択可能
	•	深い議論・実務利用向け

UIで伝えたいこと
	•	無料で試せる
	•	本格利用には APIキー設定
	•	APIキーは安全に保存
	•	議論は複数視点で進み、結論にまとまる

いまのQroomaは UI/UX はかなり見えてきているが、まだ以下が未接続 / 未完了のため、実運用状態ではありません。
	•	Supabase Auth 本接続
	•	Supabase DB 本接続
	•	Trigger.dev 本接続
	•	実際の OpenAI / Anthropic / Google API 呼び出し
	•	Free 用の固定実行ロジック
	•	BYOK 用の実行ロジック
	•	本番環境の設定と最終E2E確認

つまり、
	•	操作感や見た目は確認できる
	•	ただし 本当に議論が動く本番状態ではまだない

今回のデザインの目標

ゴール
	•	“AIサービスっぽい派手さ” を減らす
	•	“白に黒を置いただけ” の単調さもなくす
	•	静かで上質、でも平坦ではない
	•	Appleっぽい空気感 + 実用的なB2B SaaS
	•	“生成AIのLP” ではなく、思考支援プロダクト に見せる

重要な考え方

今回は「おしゃれにする」ではなく、以下を改善してください。
	•	Hierarchy（情報の主従）
	•	Density（密度）
	•	Refinement（仕上がり）
	•	Semantics（表示の意味）
	•	Reliability（UIの壊れにくさ）

直したい問題

1. まだ平坦
	•	文字サイズ、太さ、色が均一
	•	白に黒とグレーを置いただけに見える
	•	Hero / section title / card title / body / helper の差が弱い

2. まだチープ
	•	カードが全部似た見た目
	•	一覧UIがテンプレ感ある
	•	“上質なプロダクト” ではなく “作り途中の管理画面” に見える

3. まだ間延び
	•	Rooms 一覧カードが高い
	•	Settings カードが高い
	•	使い方セクションが縦に長すぎる
	•	モバイルで1画面に見える情報量が少ない

4. 右側が画面に収まっていない
	•	Rooms 一覧カード右端
	•	エラーバッジ / ステータス表示
	•	テキスト末尾や補助要素
が収まり切っていない / 切れて見える

5. CTA ボタンの文言崩れ
	•	言語切替時に Hero CTA の文言が壊れる
	•	locale 切替時の文言取得か描画に問題がある

6. チャット表示がまだモデル名寄り
	•	Qrooma はモデル名を見せる場ではなく、複数の視点で議論する場にしたい
	•	主表示は role / side ベースに寄せたい

7. 動きが少ない
	•	静かにしたいが、止まりすぎている
	•	少しだけ気持ちいい motion が欲しい

8. Settings に保存ボタンがない
	•	APIキー
	•	モード
	•	2/3エージェント数
	•	モデル設定
のような重い設定は、自動保存より 明示保存 の方が自然

⸻

デザイン修正方針

A. Typography hierarchy を強くする

最低限、以下の階層を明確に分ける
	•	Hero
	•	セクション見出し
	•	カード見出し
	•	本文
	•	補助文 / ラベル / metadata

やってほしいこと
	•	サイズ差をもっとつける
	•	weight 差をつける
	•	行間と余白でも差をつける
	•	色も primary / secondary / tertiary の3階層にする

B. Surface layering を入れる

白系の中で面の差を作る
	•	page background
	•	section surface
	•	card surface

全部同じ白にしない
差はごく薄くてよいが、静かに深さがある 状態を作る

C. LP に軽いモダンなグラデーションを入れる

方向
	•	Hero 背景や上部に、薄いブルー / ライラック / グレー系の radial gradient
	•	透明度はかなり低め
	•	ふわっと空気感が出る程度
	•	強い主張はしない
	•	白ベースの上質感を壊さない

禁止
	•	強い青紫グラデーション
	•	ネオン感
	•	CTA への強いグラデーション
	•	未来っぽい発光演出

D. カードを全部同じ見た目にしない

少なくとも以下で少し性格を分ける
	•	一覧カード
	•	設定カード
	•	情報カード
	•	結論カード

radius は揃えてよいが、
	•	padding
	•	密度
	•	面のトーン
	•	境界の出し方
は少し変える

E. Motion を少し入れる

派手な動きは禁止
静かな気持ちよさだけ入れる

入れてよい動き
	•	Hero / preview / cards の軽い fade-up
	•	hover のごく小さい浮き
	•	CTA の自然な transition
	•	アコーディオンのスムーズな開閉
	•	locale切替や state pill の滑らかな変化

禁止
	•	バウンス
	•	強いズーム
	•	ネオン
	•	サイバー感

⸻

機能 / UI の修正

1. 右端の収まりを確実に直す

以下すべてで、右側が画面幅に確実に収まるようにする
	•	Rooms 一覧カード
	•	エラーバッジ
	•	ステータス表示
	•	Settings フォーム
	•	Landing の各カード
	•	モバイル時の全体レイアウト

条件
	•	見切れない
	•	右端で切れて見えない
	•	横スクロールしない
	•	overflow-x-hidden で隠すだけではなく、内部レイアウトを調整して収める

2. CTA 文言崩れを修正

期待する状態
	•	ja / en 切替後も CTA 文言が常に正しく表示される
	•	文字化け、重なり、旧文言残りが起きない
	•	Hero CTA / Nav CTA ともに崩れない
	•	モバイルでも収まる

3. チャット中の主表示を role 名ベースにする

表示名は、役割名 + モデル名の括弧表記 にしてください。

日本語UI
	•	提案（GPT）
	•	検証（Claude）
	•	実行（Gemini）
	•	結論

英語UI
	•	Proposal (GPT)
	•	Review (Claude)
	•	Execution (Gemini)
	•	Conclusion

4. Free / BYOK の違いをUIに反映

Free
	•	APIキーなしでも使える
	•	そのまま体験可能
	•	制限があることを自然に示す

BYOK
	•	APIキーを設定するとフル機能が使える
	•	2/3 エージェント
	•	モデル選択
	•	本格利用向け

5. APIキー取得導線を追加

各 provider ごとに、API キー取得ページへ飛べる導線を置く

対象
	•	OpenAI
	•	Anthropic
	•	Google

仕様
	•	各モデル名の横、または provider セクション内に置く
	•	文言は locale 対応
	•	ja: APIキーを取得
	•	en: Get API key
	•	押すと各社の API キー取得 / 管理ページへ遷移
	•	新しいタブで開いてよい

6. 非IT向け説明 + ステップ表示

日本語
	•	QroomaでAIに議論してもらうには、各AIサービスのAPIキーが必要です。
	•	APIキーは、そのサービスを使うための利用キーのようなものです。
	•	「APIキーを取得」を押すと、各サービスの公式ページが開きます。そこでAPIキーを作成し、Qroomaに貼り付けて保存してください。

ステップ
	1.	使いたいAIサービスを選ぶ
	2.	APIキーを取得
	3.	作成したキーをコピーする
	4.	Qroomaに貼り付けて保存する

7. 2 / 3 エージェント切替
	•	UI上に明確に表示する
	•	default は 3
	•	2 のときは Side A / Side B のみ表示
	•	3 のときは Side A / Side B / Side C を表示

8. 同一モデルの重複禁止

同じ provider + model の組み合わせは重複不可

9. 使い方セクション

大きい説明カードではなく、「① APIキーを設定」型 にする

表示したい内容
	•	① APIキーを設定
	•	② ルームを作成
	•	③ テーマを投げて、議論と結論を見る

10. 手動での結論生成導線を追加
	•	ここまでをまとめる
	•	結論を出す

バックエンド未接続なら、UIモック / disabled / coming soon でもよい
ただし将来の使い方として見えるようにする

11. Settings に保存ボタンを追加

これは重要です。

方針
	•	APIキー
	•	モード
	•	2/3エージェント数
	•	モデル設定
は 明示保存 にする

UX
	•	変更があると 保存する ボタンが有効になる
	•	変更がないと disabled
	•	保存成功後に小さく 保存しました / Saved を出す
	•	自動保存前提の不安をなくす

補足
	•	UI言語切替だけは自動保存でも可
	•	それ以外の重い設定は保存ボタンがある方が自然
