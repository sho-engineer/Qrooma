Qrooma の仕上げとして、以下を修正・改善してください。
今回は プロンプトモードの一般化（旅行寄りからビジネス汎用へ） と UIの視認性改善、そして 次フェーズに向けた実装整理 が目的です。

1. プロンプトモードをもっと一般化する

現状の問題

* プロンプトモードの文言や例が旅行寄り
* 汎用の意思決定ツールというより、旅行相談向けに見える
* Qrooma の本来の価値である「ビジネス議論・比較検討・意思決定支援」が伝わりにくい

目的

プロンプトモードを、ビジネスでもそのまま使える汎用モード にする

修正方針

文言・例・プレースホルダーを、旅行ではなく 一般化されたビジネス寄り表現 に置き換えてください。

⸻

2. プロンプトモード内の文言をビジネス寄りにする

上部の問い

現状:

* 何について議論しますか？
* 旅行例が中心

変更案

* 何を検討・判断したいですか？
* 議論したいテーマを入力してください

プレースホルダー例

旅行寄りではなく、以下のような汎用例に変更してください。

推奨例

* 新機能を今期に入れるべきか検討したい
* テレワーク導入方針を決めたい
* 採用優先順位を比較したい
* 価格改定の是非を判断したい
* 営業施策の優先順位を決めたい
* 開発ロードマップの選択肢を整理したい

⸻

3. 各入力項目も一般化する

A. 議論の目的

現状でも近いが、よりビジネスで自然な粒度に揃える

表示案

* おすすめを決めたい
* 比較して優先順位を決めたい
* リスクを洗い出したい
* 実行プランを作りたい

→ これは維持でもよい
ただし説明文や例をビジネス寄りに寄せる

B. 最終的に決めたいこと

プレースホルダー例を変更
現状:

* 旅行プランを1つ決める / 候補を3つまで絞る

変更案:

* 例: 施策を1つ選ぶ / 候補を3つまで絞る / 導入方針を決める

C. 比較軸

旅行だけでなくビジネスで使いやすい軸も見せる

追加候補

* ROI
* 実装コスト
* 導入スピード
* 社内負荷
* 再現性
* インパクト
* 継続性
* 顧客価値
* オペレーション負荷
* 学習コスト

既存で残してよいもの

* コスト
* リスク
* 実行しやすさ
* 満足度

D. 制約条件

プレースホルダー例をビジネス寄りにする

変更案:

* 例: 予算100万円以内 / 2か月以内に実施 / 追加採用なし / 現体制で回す

⸻

可能なら、プロンプトモードにワンタップのテンプレートを追加してください。

例

* 施策比較
* 採用判断
* 新機能優先順位
* 価格戦略
* 組織運営
* 営業戦略
* リスクレビュー

各テンプレートで初期値を入れる

例: 施策比較

* 目的: 比較して優先順位を決めたい
* 比較軸: ROI / 実装コスト / 導入スピード / リスク

5. 「Free」の文字が見えない問題を直す

現状

* Free タブ / Free バッジ / Free ラベルの文字が背景に埋もれて見えにくい
* 視認性が悪い
* UI品質としてNG

原因候補

* selected / unselected state の文字色コントラスト不足
* 半透明背景 + 薄い文字色
* fixed bar の重なりや blur により見えづらい
* theme token の組み合わせが弱い

必須対応

A. コントラストを上げる

* Free 表示の文字色を背景と十分に差がつくようにする
* WCAG AA を意識する
* 特に selected / disabled / muted 状態でも読めるようにする

B. 状態別スタイルを見直す

* selected
* unselected
* disabled
* inactive
    を見直し、Free だけ見えにくくならないようにする

C. fixed plan switcher 全体も確認

* Free / Connect / Pro の3つで視認性が揃うようにする
* iPhoneでの白背景・グレー背景・半透明背景でも読めること

期待挙動

* Free の文字が常に読める
* selected 時も unselected 時も見える
* 下部固定バーでも埋もれない

⸻

6. ここまでで大体揃った後にやると良いこと

大きな仕様追加より、まず以下を軽く仕上げると完成度が上がります。

優先度高

1. 空状態 / ローディング / エラー状態の統一
2. 各モードのヘルプ文言整理
3. 履歴検索 / 並び替え
4. 使用量表示（Freeであと何回か）
5. 出力のコピー / 共有

優先度中

6. ルームタイトル自動補完の精度改善
7. 議論の要点だけ表示する compact view
8. 結論だけ抜き出す export

7. 画像入力に進んでよいか

結論

はい、進んでよいです。
ただし、画像をただ添付できるだけでは弱いので、Qroomaらしい画像活用 にしてください。

画像入力の方向性

以下のような使い方ができると強いです。

A. スクリーンショット議論

* LP
* UI
* 広告クリエイティブ
* 資料
* グラフ

をアップして、AIがそれを前提に議論する

B. 画像を見た上で比較

* 案A/Bのデザイン比較
* 競合LP比較
* UI改善ポイント議論

C. 画像から論点抽出

* 何が強いか
* 何が弱いか
* 改善優先順位
* どこが伝わりにくいか

最初の実装で十分な範囲

まずは以下だけでよいです。

* 画像アップロード
* 画像プレビュー
* AIが画像内容を要約
* その要約を議論の入力に組み込む

いきなり高度な画像比較やアノテーションまでやらなくてよい
